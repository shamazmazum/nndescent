How to use

Suppose you have a set of 3-dimensional points *ps* and *qs*: 

(defun make-points (n d)
  (make-array n
              :initial-contents
              (loop repeat n
                    collect
                    (make-array
                     d
                     :element-type 'single-float
                     :initial-contents (loop repeat d collect (random 1f0))))))

(defparameter *ps* (make-points 200000 3))
(defparameter *qs* (make-points 100000 3))

Now, for each query point in *qs* you want to find 2 points from *ps* which are closest to the query point according to the Euclidean metric. You can do it the slow way, using a brute-force algorithm: 

(time
 (defparameter *slow*
   (nndescent/naive:knn #'nndescent/metrics:euclidean-dist *ps* *qs* 2)))

This takes approximately 48 seconds of execution time with 6 workers (nndescent is parallelized with lparallel, so don't forget to create a new kernel before running nndescent: (setq lparallel:*kernel* (lparallel:make-kernel 6))).

Now we do the same with nndescent. Firstly we make a connectivity graph for the set *ps*, connecting each point of *ps* with the 30 closest points from this set. 

(time
 (defparameter *graph*
   (nndescent/nndescent:nndescent
    #'nndescent/metrics:euclidean-dist *ps*
    (nndescent/random-forest:initial-approximation
     #'nndescent/metrics:euclidean-dist *ps* 30)
    30 :max-iterations 20)))

This takes 6 seconds of execution time. At first we do an initial approximation of the connectivity graph with initial-approximation and then iteratively improve it with nndescent.

Now we can perform the search using the connectivity graph: 

(time
 (defparameter *fast*
   (nndescent/nndescent:knn #'nndescent/metrics:euclidean-dist *ps* *qs* *graph* 2)))

This takes 0.6 seconds. So we have approximately 48/(0.6+6)=7x speed-up compared to the brute-force algorithm.

Now you have learned how to use nndescent and may wonder what requirements it imposes on the space your points are in. As I said, unfortunately it does not work good with big dimensionalities, but virtually there is no more restrictions. For example, your metric can violate metric axioms, for example, the third axiom (the triangle inequality), so the squared Euclidean metric is also good enough for nndescent. Generally speaking, the triangle inequality is replaced by a vastly formulated requirement: if \(\rho(a,b)\) and \(\rho(a,c)\) is small, then \(\rho(b,c)\) is also small.
